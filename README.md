## 简介
基于[Fiora](https://github.com/yinxin630/fiora)进行的二次开发，将原本后端使用的[Koa](https://koajs.com/)框架替换为[Nest](https://nestjs.com/)框架。

### FAQ
- 为什么要使用Nest框架？
  - 规范化后端
  - 便于维护
  - 降低资源消耗
  - 更好的功能扩展性
- 其他需要二次开发的理由？
  - 原项目没有考虑事务问题，有潜藏风险
  - 原项目使用常量来定义项目配置，不易维护
  - 代码凌乱，不易维护
  - 没有软删除功能
